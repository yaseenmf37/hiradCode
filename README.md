# هیراد کد — سایت نمونه‌کار + پنل مدیریت

سایت نمایش نمونه‌کار با پنل مدیریت، ساخته‌شده با Next.js 16، TypeScript و Tailwind v4.
هر چیزی که لازم دارد روی **تیر رایگان** کار می‌کند — نیازی به خرید هیچ سرویسی نیست.

---

## اجرا روی سیستم خودتان

```bash
npm install
npm run dev
```

سایت روی <http://localhost:3000> بالا می‌آید و پنل روی `/admin`.

**رمز پیش‌فرض در حالت توسعه: `hirad1234`**

بدون تنظیم دیتابیس، پروژه‌ها در پوشه‌ی `.data/` روی همین سیستم ذخیره می‌شوند.
یعنی می‌توانید کل پنل را همین حالا تست کنید. این پوشه در `.gitignore` هست و
روی Vercel کار نمی‌کند (آنجا فایل‌سیستم فقط‌خواندنی است) — برای انتشار باید
دیتابیس وصل کنید.

---

## انتشار روی Vercel

### ۱. آپلود کد

پروژه را روی یک ریپوی GitHub بگذارید، بعد در [vercel.com](https://vercel.com)
گزینه‌ی **Add New → Project** و ریپو را انتخاب کنید. Vercel خودش Next.js را
تشخیص می‌دهد.

### ۲. دیتابیس (لازم)

در داشبورد پروژه روی Vercel:

**Storage → Create Database → Neon (Postgres) → Connect**

Vercel خودش متغیر `DATABASE_URL` را اضافه می‌کند. تیر رایگان Neon برای این سایت
چند برابر کافی است. جدول‌ها در اولین اجرا خودکار ساخته می‌شوند — نیازی به
migration دستی نیست.

### ۳. رمز پنل (لازم)

**Settings → Environment Variables** و این دو را اضافه کنید:

| متغیر            | مقدار                               |
| ---------------- | ----------------------------------- |
| `ADMIN_PASSWORD` | رمزی که خودتان انتخاب می‌کنید       |
| `AUTH_SECRET`    | یک رشته تصادفی طولانی (دستور پایین) |

برای ساخت `AUTH_SECRET`:

```bash
openssl rand -hex 32
```

> بدون این دو متغیر، ورود به پنل روی پروداکشن **کار نمی‌کند** — این عمدی است تا
> سایت با رمز پیش‌فرض منتشر نشود.

### ۴. آپلود تصویر (اختیاری ولی توصیه‌شده)

**Storage → Create → Blob → Connect**

متغیر `BLOB_READ_WRITE_TOKEN` خودکار اضافه می‌شود و بعد از آن می‌توانید در پنل
اسکرین‌شات‌ها را مستقیم آپلود کنید (کشیدن و رها کردن). تیر رایگان یک گیگابایت است.

بدون این، آپلود کار نمی‌کند ولی می‌توانید آدرس تصویر را دستی وارد کنید.

### ۵. آدرس سایت (اختیاری)

`NEXT_PUBLIC_SITE_URL` را روی دامنه‌ی نهایی بگذارید (مثلاً `https://hiradcode.ir`)
تا `sitemap.xml` و تگ‌های اشتراک‌گذاری درست شوند.

بعد از افزودن متغیرها یک بار **Redeploy** کنید.

---

## دامنه‌ی `.ir`

بعد از خرید دامنه از [ایرنیک](https://www.nic.ir):

1. در Vercel: **Settings → Domains → Add** و دامنه را وارد کنید.
2. در پنل ایرنیک، رکوردهایی که Vercel نشان می‌دهد را ثبت کنید
   (معمولاً یک رکورد `A` به `76.76.21.21` و یک `CNAME` برای `www`).
3. تا چند ساعت DNS پخش می‌شود و SSL خودکار صادر می‌شود.

## کار با پنل

`/admin` → ورود با `ADMIN_PASSWORD`.

- **داشبورد** — آمار کلی، آخرین پروژه‌ها و پیام‌ها
- **نمونه‌کارها** — افزودن، ویرایش، حذف، جستجو، و تیک «ویژه»
- **پیام‌ها** — پیام‌های فرم تماس سایت

هر تغییری بلافاصله روی سایت عمومی اعمال می‌شود (صفحات خودکار revalidate می‌شوند).

**پروژه‌های ویژه** بالای صفحه‌ی اصلی نمایش داده می‌شوند. اگر کمتر از سه پروژه را
ویژه کنید، جاهای خالی با جدیدترین پروژه‌ها پر می‌شود تا ردیف ناقص نماند.

---

## چند نکته

**پروژه‌های نمونه.** چهار پروژه‌ی نمایشی از قبل داخل دیتابیس ثبت می‌شوند تا سایت
از روز اول خالی نباشد. اینها را از پنل حذف کنید و پروژه‌های واقعی خودتان را
اضافه کنید. تصویرهایشان placeholder محلی هستند (`public/placeholders/`).

**تصاویر لوکال.** اگر روی شبکه‌ای هستید که DNS را فیلتر می‌کند، تصاویر با آدرس
خارجی (مثل Unsplash) روی `localhost` لود نمی‌شوند — Next آنها را به‌عنوان
private-IP بلاک می‌کند. روی Vercel مشکلی نیست. به همین دلیل تصاویر نمونه محلی‌اند.

**عنوان فارسی.** slug خودکار از عنوان ساخته می‌شود و فارسی را نگه می‌دارد
(`/works/فروشگاه-آسترا`). اگر آدرس انگلیسی می‌خواهید، فیلد slug را دستی پر کنید.

## ساختار

```
app/
  (site)/          صفحات عمومی — خانه، نمونه‌کارها، خدمات، درباره، تماس
  admin/           پنل: login عمومی، (dashboard) پشت گارد احراز هویت
  api/upload/      توکن آپلود Vercel Blob
components/
  site/ ui/ admin/
lib/
  db.ts            دو درایور: Postgres (پروداکشن) و فایل JSON (توسعه محلی)
  auth.ts          نشست تک‌کاربره با کوکی امضاشده
  projects.ts      خواندن پروژه‌ها
  seed.ts          پروژه‌های نمونه
  content.ts       متن‌های ثابت (خدمات، فرآیند، سوالات)
  site-config.ts   اطلاعات تماس و شبکه‌های اجتماعی
```

برای عوض کردن ایمیل، تلفن و شبکه‌های اجتماعی فقط `lib/site-config.ts` را ویرایش کنید.
